   b. Grep `addons/` for occurrences of this literal:
      - Use `grep -rn "{literal}" addons/` (exclude compat_nr6hal)
      - Classify each hit as WRITER (setVariable, publicVariable, =) or READER (getVariable, isNil, direct use in expressions)
      - A variable is ALIVE if it has at least ONE reader
      - A variable is DEAD if it has ZERO readers (only writers, or no occurrences at all)
   c. Also check for the GVAR/EGVAR macro form (e.g., `GVAR(activity)`, `EGVAR(core,activity)`) since many references use the macro rather than the literal expansion
   d. Also check for QGVAR/QEGVAR string forms (e.g., `QGVAR(activity)`, `QEGVAR(core,activity)`)

3. Mark each entry:
   - If ALIVE: set `"stripped": false` (keep existing value)
   - If DEAD: set `"stripped": true`

4. Write the updated rename-map.json back

5. Print summary:
   - Total entries: 506
   - Living (stripped=false): N
   - Dead (stripped=true): N
   - List the dead variable names for review

**Important considerations:**
- Some variables may appear only in the compat addon (compat_nr6hal) — those references do NOT count as readers. Exclude compat_nr6hal from the grep.
- Some variables are set in one addon and read in another via publicVariable/getVariable — the reader may use QEGVAR form. Check all addon directories.
- Variables that are ONLY publicVariable'd but never getVariable'd elsewhere may be dead (broadcast with no listener).
- Be conservative: if in doubt, mark as ALIVE (false positive is safe, false negative breaks things).
