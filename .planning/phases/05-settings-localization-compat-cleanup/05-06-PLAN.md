1. **Create `addons/core/initSettings.inc.sqf`** with CBA_fnc_addSetting calls for all settings enumerated in the research (05-RESEARCH.md Block 2).

Use the ACE3 standard pattern for each setting:
```sqf
[
    QEGVAR(core,smoke),
    "CHECKBOX",
    [LLSTRING(smoke), LLSTRING(smoke_desc)],
    LLSTRING(category_behaviour),
    true,
    1
] call CBA_fnc_addSetting;
```

**Setting categories** (use LLSTRING for localized names):
- "HAL General" — 26 settings from GenSettings_Module
- "HAL Commander" — 15 settings from Leader_Settings_Module
- "HAL Behaviour" — 23 settings from Leader_BehSettings_Module
- "HAL Personality" — 2 settings from Leader_PersSettings_Module
- "HAL Support" — 11 settings from Leader_SupSettings_Module
- "HAL Objectives" — 21 settings from Leader_ObjSettings_Module
- "HAL High Commander" — 3 settings from BBSettings_Module

**Setting types:**
- CHECKBOX: Boolean settings (true/false)
- SLIDER: Numeric settings — specify [min, max, default, step]. Read the CfgVehicles.hpp argument definitions for the actual min/max/step values (they're in the `class Attributes` `control` class).
- LIST: Enum settings (e.g., aIChat_Type with values "NONE"/"sideChat"/"groupChat", personality with values "COMPETENT"/"RECKLESS"/etc.)

**Scope parameter (6th argument):**
- `1` (server setting, publicVariable'd) for settings that affect AI behavior globally
- `0` (client setting) for UI-only settings like infoMarkers, actions visibility
- When in doubt, use `1` (server) — AI behavior settings must be server-authoritative

**Important:** The setting variable names MUST match the EGVAR/GVAR names already used in the module functions (from Phase 4 renames). Use the exact names from the CfgVehicles.hpp argument class definitions. For example, if CfgVehicles.hpp defines `class EGVAR(core,smoke)`, the setting name is `QEGVAR(core,smoke)`.

Note: initSettings.inc.sqf lives in addons/core/ but uses LLSTRING which resolves relative to the core component. The stringtable keys must use the `STR_hal_core_*` prefix for core-component settings.

1. **Update fnc_leader*Settings.sqf default fallbacks:**

In each of the 6 fnc_leader*Settings.sqf functions, find lines like:
```sqf
missionNamespace setVariable [QEGVAR(core,smoke) + _letter, _logic getVariable [QEGVAR(core,smoke), true]];
```

Change the hardcoded default (`true`) to the CBA setting value:
```sqf
missionNamespace setVariable [QEGVAR(core,smoke) + _letter, _logic getVariable [QEGVAR(core,smoke), EGVAR(core,smoke)]];
```

This way:
- If module is placed with explicit attribute -> module value used
- If module is placed without attribute -> CBA setting default used
- If no module at all -> CBA setting is already set globally during preInit

Do this for ALL settings in each function:
- fnc_generalSettings.sqf: ~26 settings
- fnc_leaderSettings.sqf (Commander): ~15 settings
- fnc_leaderBehaviourSettings.sqf: ~23 settings
- fnc_leaderPersSettings.sqf: ~2 settings
- fnc_leaderSupSettings.sqf: ~11 settings
- fnc_leaderObjSettings.sqf: ~21 settings
